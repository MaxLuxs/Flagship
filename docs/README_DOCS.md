# Документация Flagship

## Генерация HTML из Markdown

Все HTML файлы в папке `docs/` автоматически генерируются из соответствующих Markdown файлов.

### Как это работает

1. **Источник истины**: Все содержимое находится в `.md` файлах (например, `MIGRATION_GUIDE.md`)
2. **Автоматическая генерация**: HTML файлы (например, `MIGRATION_GUIDE.html`) генерируются из Markdown
3. **Единый стиль**: Все HTML файлы используют единый шаблон с одинаковым дизайном

### Команды

```bash
# Сгенерировать все HTML файлы из Markdown
./gradlew generateDocsHtml

# Или просто обновить один файл - измените .md файл и запустите:
./gradlew generateDocsHtml
```

### Правила работы

- ✅ **Редактируйте только `.md` файлы** - это источник истины
- ✅ **Не редактируйте `.html` файлы вручную** - они будут перезаписаны при следующей генерации
- ✅ **Коммитьте оба файла** (`.md` и `.html`) - HTML нужен для GitHub Pages
- ✅ **Проверяйте результат** после изменений: `./gradlew generateDocsHtml`

### Поддерживаемые Markdown элементы

- Заголовки (`#`, `##`, `###`, `####`)
- Параграфы
- Списки (нумерованные и маркированные)
- Блоки кода (```)
- Inline код (`code`)
- Ссылки `[text](url)`
- Изображения `![alt](url)`
- Жирный текст `**bold**`
- Курсив `*italic*`
- Цитаты `> quote`
- Горизонтальные линии `---`

### CI/CD

HTML файлы автоматически генерируются в GitHub Actions при каждом push в `main` ветку.
